# Part 3.1:codex 多账户

> 在 Part 3 的单账户 codex OAuth 基础上,支持登录多个 ChatGPT 账户、在它们之间切换。全局单一"当前账户"(非 per-project)。

## 背景

Part 3 实现了 codex OAuth,但 token 存在 SecretStorage 单 key、只保留一个账户,登录新账户会覆盖旧的。本轮支持多账户:同时保存多个账户的凭证,选一个作为"当前账户"用于转发。

## 范围

- **做**:登录多个 codex 账户、列出/切换/登出各账户、转发用当前账户、token 各自刷新、UI 用 email 区分。
- **不做**:per-project 当前账户(全局单一)、账户级别的额度/用量展示。

## 存储(全局 SecretStorage)

`claudeProxy.codex` 的值从单 `StoredToken` 改为:
```ts
interface CodexStore {
  accounts: Record<string, StoredToken>; // key = accountId
  current: string;                        // 当前账户 accountId('' 表示无)
}
interface StoredToken {
  accessToken: string;
  refreshToken: string;
  accountId: string;
  email: string;       // 新增,显示用
  expiresAt: number;
}
```
读取时若解析出的不是 `CodexStore` 形状(旧的单账户格式),视为无账户(不迁移,重新登录即可)。

## oauth.ts

- 新增 `parseAccountInfo(idToken): { accountId: string; email: string }`(替代/扩展现有 `parseAccountId`):从 JWT claims 取 `https://api.openai.com/auth`.chatgpt_account_id 与 `email`。
- `buildAuthUrl` 已含 `prompt=login`,保证添加账户时浏览器可登录不同账号。

## auth.ts(CodexAuth 重构为多账户)

| 方法 | 行为 |
|---|---|
| `addAccount(token)` | 解析 accountId/email,存入 `accounts[accountId]`,设 `current = accountId` |
| `listAccounts()` | 返回 `{ accountId, email, isCurrent }[]` |
| `setCurrent(accountId)` | 设 current(仅当该账户存在) |
| `removeAccount(accountId)` | 从 accounts 删除;若删的是 current,则 current 切到任一剩余账户或 `''` |
| `isLoggedIn()` | accounts 非空 |
| `getValid()` | 取 **current** 账户 token;过期则 refresh 并回存;无 current 或刷新失败返回 null |

`getValid()` 签名不变(`Promise<{accessToken, accountId} | null>`),proxy 无需改动。

## login.ts

`loginCodex(auth)` 改为成功后调 `auth.addAccount(tok)`(而非旧的 `save`),支持多次登录累加账户。

## statusbar.ts

Provider 设置 → codex 项进入后(替换原"登录/登出"二选一):
- 若有账户:列出每个 `<email>`(当前账户前缀 `$(check)`),点击 → `setCurrent`;
- 「$(add) 添加账户」→ 执行登录命令(loginCodex);
- 每个账户可「$(trash) 登出 <email>」→ `removeAccount`。

codex 在状态栏一级菜单作为可切换 provider 的条件不变(`isLoggedIn()` 为真即出现)。

## proxy / extension

- `proxy.ts`:不变(codex 分支用 `deps.getCodexAuth()` = `getValid()`,内部已切到 current 账户)。
- `extension.ts`:`codexLogin` 命令不变(loginCodex 内部改为 addAccount);`codexLogout` 命令语义变为"登出当前账户"或保留为清空——本轮让 UI 的逐账户登出承担,`codexLogout` 命令改为登出 current 账户(供命令面板用)。

## 测试

- `auth.ts`:把账户管理逻辑(增/删/切 current、删 current 后的回退)抽成可对注入的 fake `SecretStorage` 测试 —— 覆盖 addAccount 设 current、setCurrent、removeAccount 删非当前/删当前的回退、isLoggedIn。
- `oauth.ts`:`parseAccountInfo` 从 id_token 取 accountId + email;坏 token 返回空。
- UI(statusbar)/登录流走 F5 手动冒烟(登录两个账号、切换、各自登出)。

## 范围外

- per-project 当前账户、用量/额度展示、账户重命名。
